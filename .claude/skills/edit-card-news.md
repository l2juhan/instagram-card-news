# 카드뉴스 수정 파이프라인

이미 생성된 카드뉴스(`workspace/slides.json`)를 수정하고 다시 렌더링합니다.

## Step 1: 현재 상태 확인

1. `workspace/slides.json`을 읽어 현재 슬라이드 구성을 파악합니다
2. `output/` 디렉토리의 기존 PNG 파일들을 확인합니다
3. 사용자의 수정 요청을 분석합니다

**slides.json이 없는 경우**: 사용자에게 먼저 카드뉴스를 생성하라고 안내합니다.

---

## Step 2: 수정 유형 판별

사용자의 요청을 다음 유형으로 분류합니다:

| 유형 | 예시 | 처리 방식 |
|---|---|---|
| **텍스트 수정** | "3번 슬라이드 헤드라인 바꿔줘" | 해당 필드만 직접 수정 |
| **슬라이드 타입 변경** | "4번을 리스트 형식으로 바꿔줘" | type과 필드 구조를 변경 |
| **슬라이드 추가** | "통계 슬라이드 하나 추가해줘" | 지정 위치에 새 슬라이드 삽입, slide 번호 재정렬 |
| **슬라이드 삭제** | "5번 슬라이드 빼줘" | 해당 슬라이드 제거, slide 번호 재정렬 |
| **슬라이드 순서 변경** | "3번이랑 5번 순서 바꿔줘" | 순서 변경 후 slide 번호 재정렬 |
| **스타일 변경** | "볼드 스타일로 바꿔줘" | 템플릿 스타일만 변경 (slides.json 유지) |
| **악센트 색상 변경** | "악센트 색상 빨간색으로" | 렌더링 시 --accent 옵션만 변경 |
| **전체 재작성** | "전체적으로 톤을 캐주얼하게" | Step 3으로 이동하여 카피 재작성 |
| **특정 슬라이드 재작성** | "3번 내용을 더 자세하게" | 해당 슬라이드만 리서치 참고하여 재작성 |

---

## Step 3: 수정 실행

### 단순 수정 (텍스트/타입/추가/삭제/순서)

직접 `workspace/slides.json`을 수정합니다:

1. 현재 slides.json을 읽음
2. 요청에 따라 수정
3. **slide 번호 재정렬** (1부터 순서대로)
4. 수정된 slides.json을 저장
5. Step 4로 이동

### 카피 재작성 (전체/부분)

서브 에이전트를 사용하여 카피를 재작성합니다:

**에이전트**: general-purpose
**모델**: sonnet

- `workspace/research.md` (기존 리서치)와 `workspace/slides.json` (현재 슬라이드)을 입력으로 전달
- 사용자의 수정 피드백을 구체적으로 전달
- 수정 대상 슬라이드만 재작성 (나머지 유지)
- 결과를 `workspace/slides.json`에 저장

### 스타일/악센트 변경

slides.json은 수정하지 않고 Step 4에서 렌더링 옵션만 변경합니다.

---

## Step 4: 렌더링

수정된 slides.json을 다시 렌더링합니다.

### 전체 렌더링

스타일 변경, 전체 재작성, 다수 슬라이드 수정 시:

```bash
node scripts/render.js \
  --slides workspace/slides.json \
  --style {template} \
  --output output/ \
  --accent "{accent_color}" \
  --account "{account_name}"
```

### 부분 렌더링

특정 슬라이드만 수정된 경우, `--only` 옵션으로 해당 슬라이드만 렌더링:

```bash
node scripts/render.js \
  --slides workspace/slides.json \
  --style {template} \
  --output output/ \
  --accent "{accent_color}" \
  --account "{account_name}" \
  --only 3,5
```

> `--only` 옵션이 render.js에서 지원되지 않으면 전체 렌더링으로 대체합니다.

---

## Step 5: 수정 결과 검토

렌더링 완료 후:

1. 수정된 슬라이드의 PNG를 읽어서 결과를 확인합니다
2. 수정 요청이 정확히 반영되었는지 검증합니다
3. 텍스트 잘림, 레이아웃 깨짐 등 문제가 없는지 확인합니다

**문제 발견 시**: Step 3으로 돌아가 재수정
**이상 없음**: 사용자에게 수정 완료 보고 및 변경 내용 요약

---

## 참고 사항

- 수정 전 항상 기존 `workspace/slides.json`을 먼저 읽어 현재 상태를 파악합니다
- 사용자가 여러 수정을 한 번에 요청하면 모두 반영한 후 한 번만 렌더링합니다
- `cta` 타입 슬라이드는 생성하지 않습니다 (프로젝트 규칙)
- 빈 공간이 50% 이상인 슬라이드가 생기지 않도록 주의합니다
- 스타일 변경 시 해당 `/style-{template}` 스킬을 호출하여 가이드를 로드합니다
- `code_body` 필드는 HTML 태그가 포함될 수 있으므로 수정 시 주의합니다
- `body` 필드에서 줄바꿈은 `\n`을 사용합니다 (render.js가 `<br>`로 변환)
